## 三、订单状态流转检查

### 3.1 状态定义

| 状态码 | 状态 | 说明 |
|--------|------|------|
| 0 | 待支付 | 订单已创建，等待支付 |
| 1 | 待接单 | 支付成功，等待商户接单 |
| 2 | 制作中 | 商户已接单，正在制作 |
| 3 | 已完成 | 订单完成 |
| 4 | 已取消 | 订单已取消 |
| 5 | 已退款 | 订单已退款 |

### 3.2 状态流转规则

```javascript
{
  0: [1, 4],       // 待支付 -> 待接单/已取消
  1: [2, 4, 5],    // 待接单 -> 制作中/已取消/已退款
  2: [3, 4, 5],    // 制作中 -> 已完成/已取消/已退款
  3: [],           // 已完成 -> 不可变更
  4: [],           // 已取消 -> 不可变更
  5: []            // 已退款 -> 不可变更
}
```

### 3.3 状态流转图

```
待支付(0) ──支付成功──→ 待接单(1) ──接单──→ 制作中(2) ──完成──→ 已完成(3)
    │                     │                  │
    └──────取消──────→ 已取消(4) ←────取消───┘
                          ↑                  │
                          └──────退款──────→ 已退款(5)
```
