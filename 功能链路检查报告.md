## 问题详情与修复

### 1. 云函数调用问题

**问题描述：**
前端代码调用云函数时使用了旧的独立函数名（如 `createPayment`、`cancelOrder`），但云函数已整合为模块化设计。

**修复内容：**
| 文件 | 旧调用 | 新调用 |
|------|--------|--------|
| pages/order/order.js | `name: 'createPayment'` | `name: 'order', action: 'pay'` |
| pages/order/order.js | `name: 'cancelOrder'` | `name: 'order', action: 'cancel'` |
| pages/order/order.js | `name: 'getOrderDetail'` | `name: 'order', action: 'getDetail'` |
| packageOrder/orderDetail/orderDetail.js | 同上 | 同上 |
| packageUser/addressEdit/addressEdit.js | `name: 'addAddress'` | `name: 'address', action: 'add'` |

---

### 2. 模拟数据问题

**问题描述：**
`utils/mock.js` 的 `isDevMode()` 在开发者工具中返回 `true`，导致商户后台使用模拟数据，订单 ID 是假的（如 `order_001`），无法进行实际操作。

**修复方案：**
清空模拟数据，强制使用真实云函数数据。

```javascript
// utils/mock.js
function isDevMode() {
  return false
}

var mockOrders = []
var mockDishes = []
var mockTables = []
var mockCategories = []
```

---

### 3. 订单删除限制

**问题描述：**
1. 订单删除条件过于严格
2. 商户端订单卡片缺少删除按钮
3. 用户端没有删除订单功能

**修复内容：**
- 商户可删除任何状态订单
- 用户可删除待支付/已完成/已取消/已退款订单
- 所有状态都显示删除按钮

---

### 4. 订单状态优化

**问题描述：**
"已出餐"和"已完成"状态功能重复，流程冗余。

**修复方案：**
将 7 个状态合并为 6 个状态。

| 旧状态 | 新状态 |
|--------|--------|
| 0 待支付 | 0 待支付 |
| 1 待接单 | 1 待接单 |
| 2 制作中 | 2 制作中 |
| 3 已出餐 | ❌ 删除 |
| 4 已完成 | 3 已完成 |
| 5 已取消 | 4 已取消 |
| 6 已退款 | 5 已退款 |

**涉及文件：**
- cloudfunctions/order/index.js
- utils/merchant-api.js
- packageMerchant/merchantHome/merchantHome.js
- packageMerchant/merchantHome/components/order-card/
- pages/order/order.js
- packageOrder/orderDetail/orderDetail.js

---

### 5. 打印功能

**问题描述：**
接单后没有自动触发打印后厨单。

**修复方案：**
在 `handleOrderAccept` 中添加打印逻辑：

```javascript
async handleOrderAccept(e) {
  // ... 接单逻辑
  
  // 自动打印后厨单
  await this.printKitchenTicket(order)
}
```

### 6. 初始化数据

**问题描述：**
测试订单缺少必要的关联数据：
- `_openid` 是假的
- `tableId` 是空的
- `dishId` 是假的

**修复方案：**
`initDatabase` 云函数现在会：
1. 使用当前用户的真实 openid
2. 关联真实的桌号 ID
3. 关联真实的菜品 ID
