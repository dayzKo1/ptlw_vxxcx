## 二、功能链路检查

### 1. 用户登录流程 ✅

```
前端: pages/login/login.js
    ↓
云函数: user (action: login)
    ↓
返回: { openid, role, isMerchant, nickName, avatarUrl }
    ↓
存储: wx.setStorageSync('userInfo', userData)
    ↓
跳转: 商户 → merchantHome / 用户 → index
```

**修复**: 添加 `isMerchant` 字段返回并存储

---

### 2. 点餐流程 ✅

```
前端: pages/menu/menu.js
    ↓
直接查询: dishes, categories (数据库)
    ↓
前端: packageOrder/cart/cart.js
    ↓
云函数: order (action: create)
    ↓
创建订单 + 锁定桌号
```

**数据流**:
- 菜单页直接查询数据库（无权限问题，公开数据）
- 创建订单时携带 `userInfo` 用于冗余存储

---

### 3. 支付流程 ⚠️

```
前端: order.js / orderDetail.js
    ↓
云函数: order (action: pay)
    ↓
返回支付参数 (模拟)
    ↓
调用 wx.requestPayment (需配置微信支付)
    ↓
支付回调: paymentCallback
```

**状态**: 模拟支付可用，真实微信支付需配置商户号

---

### 4. 订单管理流程 ✅

**用户端**:
```
getUserList → 获取自己的订单
getDetail → 获取订单详情
cancel → 取消订单 (未支付直接取消，已支付自动退款)
```

**商户端**:
```
getMerchantList → 获取所有订单 (支持日期筛选)
updateStatus → 更新订单状态 (接单/出餐/完成)
refund → 退款
delete → 删除订单
```

**修复**: 添加 `startDate/endDate` 参数支持日期筛选

---

### 5. 商户管理流程 ✅

```
权限验证: merchantWhitelist.where({ openid, status: 1 })

统计数据: stats
菜品管理: dish (list/create/update/delete/toggle)
分类管理: dish (categoryList/categoryCreate/categoryUpdate/categoryDelete)
桌号管理: table (list/create/update/delete/toggle/generateQR/batchGenerateQR)
店铺设置: shop (get/update/toggleAutoAccept)
```

**修复**: 添加分类管理功能到 dish 云函数

---

### 6. 地址管理流程 ✅

```
list → 获取用户地址列表
add → 添加地址
update → 更新地址
delete → 删除地址
setDefault → 设置默认地址
```

**权限**: 用户只能管理自己的地址 (`_openid`)
