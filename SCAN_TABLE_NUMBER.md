# 扫码获取桌号功能说明

## 📋 功能概述

用户通过扫描桌号二维码进入小程序时，系统会自动获取桌号并保存到本地存储，方便后续点餐使用。

---

## 🔄 实现原理

### 1. 二维码生成

**云函数**: `generateTableQRCode`

生成的小程序码包含场景参数：
```javascript
{
  scene: `table=${tableNumber}`,
  page: 'pages/index/index'
}
```

**示例**：
- 1号桌的二维码：`scene=table=1`
- 2号桌的二维码：`scene=table=2`

### 2. 扫码进入小程序

**app.js** 中解析场景参数，提取桌号并保存。

### 3. 页面显示桌号

- **首页**：`pages/index/index`
- **购物车**：`packageOrder/cart/cart`

---

## 🧪 测试方法

### 开发者工具测试

1. 点击"编译"旁边的下拉菜单
2. 选择"添加编译模式"
3. 填写模式名称，如"桌号1"
4. 启动参数填写：`scene=table=1`
5. 点击"编译"

### 预期结果

- 控制台日志显示桌号已保存
- 首页显示当前桌号
- 购物车页面显示桌号
- 提交订单时桌号正确

---

## 📊 数据流程

```
用户扫码
    ↓
微信识别小程序码
    ↓
小程序启动，传入 scene 参数
    ↓
app.js 解析 scene 参数
    ↓
提取 tableNumber
    ↓
保存到本地存储和全局数据
    ↓
各页面读取并显示
```

---

## ⚠️ 注意事项

### 场景参数格式

确保场景参数格式正确：
- ✅ 正确：`table=1`
- ✅ 正确：`table=10`
- ❌ 错误：`table=1&other=value`

### 清除桌号

如需清除桌号：
```javascript
wx.removeStorageSync('tableNumber')
```
