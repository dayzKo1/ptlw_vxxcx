## 第一步：小程序注册与认证

### 1.1 注册小程序

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击「立即注册」→ 选择「小程序」
3. 填写账号信息（使用个体工商户或企业信息）
4. 完成邮箱激活

### 1.2 完成认证

1. 登录小程序后台
2. 进入「设置」→「基本设置」
3. 点击「微信认证」
4. 支付 300 元认证费
5. 上传营业执照等材料
6. 等待审核（1-3 个工作日）

### 1.3 获取 AppID

1. 认证通过后，进入「开发」→「开发设置」
2. 复制 AppID，后续配置需要使用

## 第二步：项目配置

### 2.1 安装开发者工具

1. 访问 [微信开发者工具下载页面](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 下载并安装对应系统版本

### 2.2 导入项目

1. 打开微信开发者工具
2. 点击「导入项目」
3. 选择项目目录
4. 填入 AppID
5. 点击「导入」

### 2.3 修改配置文件

修改 `project.config.json`：

```json
{
  "appid": "你的小程序AppID"
}
```

### 2.4 创建云开发环境

1. 点击工具栏「云开发」按钮
2. 点击「创建环境」
3. 选择「按量付费」（免费额度足够使用）
4. 填写环境名称，如「production」
5. 等待环境创建完成（约 1-2 分钟）
6. 复制环境 ID

修改 `cloudbaserc.json`：

```json
{
  "envId": "你的云开发环境ID"
}
```

## 第三步：云函数部署

### 3.1 云函数列表

本项目包含 24 个云函数：

| 分类 | 云函数 | 说明 |
|-----|-------|------|
| 用户 | login | 用户登录 |
| 订单 | createOrder | 创建订单 |
| 订单 | createPayment | 创建支付 |
| 订单 | paymentCallback | 支付回调 |
| 订单 | cancelOrder | 取消订单 |
| 订单 | cancelTimeoutOrders | 取消超时订单（定时任务） |
| 订单 | getUserOrders | 获取用户订单 |
| 订单 | getOrderDetail | 获取订单详情 |
| 订单 | getMerchantOrders | 获取商家订单 |
| 订单 | updateOrderStatus | 更新订单状态 |
| 菜品 | manageDish | 菜品管理 |
| 桌号 | getTables | 获取桌号列表 |
| 桌号 | manageTable | 桌号管理 |
| 桌号 | generateTableQRCode | 生成桌号二维码 |
| 桌号 | batchGenerateTableQRCode | 批量生成二维码 |
| 地址 | getAddresses | 获取地址列表 |
| 地址 | addAddress | 添加地址 |
| 地址 | updateAddress | 更新地址 |
| 地址 | deleteAddress | 删除地址 |
| 地址 | setDefaultAddress | 设置默认地址 |
| 店铺 | manageShop | 店铺信息管理 |
| 商家 | merchantStats | 商家统计 |
| 商家 | addMerchantWhitelist | 添加商家白名单 |
| 数据库 | initDatabase | 初始化数据库 |

### 3.2 部署云函数

**方式一：逐个部署**

1. 在微信开发者工具左侧目录找到 `cloudfunctions` 文件夹
2. 右键点击每个云函数文件夹
3. 选择「上传并部署：云端安装依赖」
4. 等待部署完成

**方式二：批量部署**

1. 右键点击 `cloudfunctions` 文件夹
2. 选择「同步云函数列表」
3. 等待所有云函数部署完成

### 3.3 验证部署

1. 进入云开发控制台
2. 点击「云函数」
3. 确认所有云函数状态为「部署成功」

## 第四步：数据库初始化

### 4.1 数据库集合

需要创建以下集合：

| 集合名 | 说明 | 权限 |
|-------|------|------|
| categories | 菜品分类 | 仅创建者可写 |
| dishes | 菜品 | 仅创建者可写 |
| orders | 订单 | 仅创建者可写 |
| tables | 桌号 | 仅创建者可写 |
| shopInfo | 店铺信息 | 仅创建者可写 |
| addresses | 收货地址 | 仅创建者可写 |
| orderCounters | 订单计数器 | 仅创建者可写 |
| merchantWhitelist | 商家白名单 | 仅创建者可写 |
| users | 用户信息 | 仅创建者可写 |
| config | 系统配置 | 仅创建者可写 |

### 4.2 初始化方式

**方式一：使用 initDatabase 云函数（推荐）**

在微信开发者工具调试器控制台执行：

```javascript
wx.cloud.callFunction({
  name: 'initDatabase'
}).then(res => {
  console.log('初始化结果：', res)
})
```

**方式二：手动创建**

1. 进入云开发控制台
2. 点击「数据库」
3. 点击「添加集合」
4. 逐个创建上述集合

### 4.3 验证数据

初始化后会创建以下示例数据：

- 4 个菜品分类（热菜、凉菜、主食、饮品）
- 7 个示例菜品
- 20 个桌号（1-20号）
- 店铺默认信息

## 第五步：支付配置

> ⚠️ 支付功能需要个体工商户或企业资质，个人账号无法开通

### 5.1 申请微信支付商户号

1. 访问 [微信支付商户平台](https://pay.weixin.qq.com/)
2. 点击「成为商家」
3. 填写商户信息
4. 上传营业执照、食品经营许可证等材料
5. 等待审核（1-3 个工作日）

### 5.2 关联小程序和商户号

1. 登录小程序后台
2. 进入「功能」→「微信支付」
3. 点击「关联商户号」
4. 等待商户号确认关联

### 5.3 配置云支付

1. 进入云开发控制台
2. 点击「设置」→「全局设置」
3. 找到「微信支付配置」
4. 填写商户号和 API 密钥（v2 版本）

### 5.4 修改云函数配置

修改 `cloudfunctions/createPayment/index.js` 第 35 行：

```javascript
const MCH_ID = 'YOUR_MCH_ID'  // 替换为你的商户号
```

修改后重新部署 `createPayment` 云函数。

### 5.5 测试支付

1. 在小程序中下单
2. 使用沙箱环境测试支付功能
3. 确认支付回调正常

## 第六步：商家权限配置

### 6.1 获取用户 OpenID

1. 在小程序中登录（使用开发者工具或真机）
2. 进入云开发控制台
3. 打开 `users` 集合
4. 找到对应记录，复制 `_openid` 字段

### 6.2 添加商家白名单

在云开发控制台打开 `merchantWhitelist` 集合，添加记录：

```json
{
  "openid": "用户的openid",
  "status": 1,
  "createTime": 1700000000000
}
```

或在微信开发者工具控制台执行：

```javascript
wx.cloud.callFunction({
  name: 'addMerchantWhitelist',
  data: {
    openid: '用户的openid'
  }
}).then(res => {
  console.log('添加结果：', res)
})
```

### 6.3 验证商家权限

1. 重新登录小程序
2. 进入「我的」页面
3. 应该能看到「商家管理」入口
4. 点击进入商家后台

## 第八步：上线发布

### 8.1 上传代码

1. 在微信开发者工具中点击「上传」
2. 填写版本号（如 1.0.0）
3. 填写项目备注
4. 等待上传完成

### 8.2 提交审核

1. 登录小程序后台
2. 进入「版本管理」
3. 点击「提交审核」
4. 填写审核信息：
   - 选择服务类目：餐饮服务 → 点餐
   - 填写测试账号（如需要）
   - 上传截图
5. 等待审核（1-7 个工作日）

### 8.3 审核注意事项

- 确保有实际的菜品和分类数据
- 确保支付功能正常
- 准备隐私政策页面（可在「我的」→「关于」中添加）
- 确保无敏感词汇

### 8.4 发布上线

审核通过后：

1. 进入「版本管理」
2. 点击「发布」
3. 选择「全量发布」
4. 小程序正式上线

## 第九步：运营配置

### 9.1 生成桌号二维码

1. 使用商家账号登录小程序
2. 进入「桌号管理」
3. 点击「批量生成二维码」
4. 等待生成完成
5. 逐个下载并打印

**二维码规则：**

```
跳转路径：pages/login/login?table=桌号
示例：pages/login/login?table=5
```

### 9.2 配置自动接单

1. 进入商家后台
2. 点击顶部「自动接单」开关
3. 开启后支付成功自动变为「制作中」状态

### 9.3 配置定时任务（可选）

配置订单超时自动取消：

1. 进入云开发控制台
2. 点击「云函数」→「cancelTimeoutOrders」
3. 点击「定时触发器」标签
4. 添加触发器：
   - 触发类型：定时触发
   - Cron 表达式：`*/5 * * * * *`（每 5 分钟）
5. 保存配置

### 9.4 配置蓝牙打印（可选）

1. 准备蓝牙热敏打印机
2. 在商家后台点击「打印机设置」
3. 搜索并连接打印机
4. 测试打印

## 常见问题

### Q1: 云函数部署失败

**原因：** 网络问题或环境配置错误

**解决方案：**
1. 检查网络连接
2. 确认 `cloudbaserc.json` 中的 `envId` 正确
3. 重新部署失败的云函数

### Q2: 支付功能无法使用

**原因：** 商户号未配置或未关联

**解决方案：**
1. 确认已申请微信支付商户号
2. 确认小程序已关联商户号
3. 确认 `createPayment` 云函数中 `MCH_ID` 已修改
4. 确认云开发控制台已配置支付参数

### Q3: 商家后台无法访问

**原因：** 用户未添加到白名单

**解决方案：**
1. 确认用户已登录
2. 在 `merchantWhitelist` 集合添加用户 openid
3. 重新登录小程序

### Q4: 订单提交失败

**原因：** 数据库集合未创建或云函数未部署

**解决方案：**
1. 确认所有数据库集合已创建
2. 确认 `createOrder` 云函数已部署
3. 查看云函数日志排查错误

### Q5: 图片无法显示

**原因：** 图片未上传或路径错误

**解决方案：**
1. 上传图片到云存储
2. 更新数据库中的图片路径为云存储地址

### Q6: 订单超时取消不生效

**原因：** 定时触发器未配置

**解决方案：**
1. 确认 `cancelTimeoutOrders` 云函数已部署
2. 配置定时触发器（每 5 分钟执行）

### Q7: 二维码无法生成

**原因：** 云函数权限或网络问题

**解决方案：**
1. 确认 `generateTableQRCode` 云函数已部署
2. 确认小程序已认证
3. 查看云函数日志排查错误
